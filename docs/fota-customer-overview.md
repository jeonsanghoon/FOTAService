# FOTA 업데이트 서비스 — 고객용 개요

Firmware Over-The-Air 업데이트 서비스로 디바이스 펌웨어를 안전하고 효율적으로 관리할 수 있습니다.

---

## 1. 개요

- **서비스명**: FOTA(Firmware Over-The-Air) 업데이트 서비스
- **구성**: 기존 시스템과 분리된 별도 페이지로 제공
- **인증**: 기존 시스템 로그인 활용 — AWS Cognito 기반 SSO

---

## 2. 타시스템 연동 · Cognito SSO 자동 로그인

타시스템(기존 업무 시스템)에 **FOTA 메뉴**를 두고, **Cognito SSO** 인증을 통해 우리 FOTA 플랫폼에 **자동 로그인**되는 프로세스를 지원합니다.

| 단계 | 내용 |
|------|------|
| 1. 타시스템에서 FOTA 메뉴 진입 | ERP, IoT 플랫폼 등에 구성된 FOTA 메뉴 클릭 → FOTA 플랫폼 진입 URL로 이동 |
| 2. Cognito SSO 인증 | 타시스템과 FOTA가 동일 Cognito 사용자 풀 사용. 이미 로그인된 경우 토큰으로 추가 로그인 없이 인증 |
| 3. FOTA 플랫폼 자동 로그인 | 인증 완료 후 우리 플랫폼에 자동 로그인, 대시보드·업데이트 관리 등 즉시 이용 가능 |

- 타시스템에 FOTA 전용 메뉴만 추가하면, 사용자는 한 번의 로그인(타시스템)으로 FOTA까지 동일 계정으로 이용할 수 있습니다.

---

## 3. 관리 기능

고객·지사·사용자·디바이스를 체계적으로 관리할 수 있습니다.

| 구분 | 설명 |
|------|------|
| **고객 관리** | 고객사(업체) 단위 등록·조회·수정, 담당자 및 연락처 관리 |
| **지사 관리** | 고객별 지사(사업장) 등록, 주소·코드·디바이스 수 관리 |
| **사용자 관리** | 콘솔 접근 사용자 등록, 역할(관리자/운영자/조회자) 및 권한 관리 |
| **디바이스 관리** | 디바이스 등록·목록·상태 조회, 펌웨어 버전 및 업데이트 이력 관리 |

---

## 4. 주요 기능

- **제품·버전 관리**: 제품 단위로 펌웨어 버전 등록·수정, S3 멀티파트 업로드로 대용량 펌웨어 관리
- **업데이트 유형**: 제품 그룹별 업데이트, 업체별·지사별·제품별 일괄 업데이트
- **업데이트 현황·제어**: 업체/지사/제품별 진행 현황 조회, 진행 중 업데이트 취소
- **디바이스 프로비저닝**: 디바이스 등록·인증 정보 발급, Shadow 연동 자동화

---

## 5. 서버리스 Lambda 처리 프로세스

AWS Lambda를 활용해 업로드·배포·로그 수집 등을 서버 없이 처리합니다.

| 단계 | 처리 내용 |
|------|-----------|
| **펌웨어 업로드** | S3 멀티파트 업로드 완료 시 Lambda 트리거 → 메타데이터 검증·DB 반영(제품·버전)·CloudFront 무효화 등 |
| **업데이트 Job 생성·배포** | 사용자가 배포 실행 시 API Gateway → Lambda → IoT Jobs 등록 및 Shadow desired 반영, MQTT로 디바이스 전파 |
| **Presigned URL 발급** | 디바이스가 만료된 다운로드 URL 요청 시 Lambda가 S3 presigned URL 생성 후 반환 |
| **로그·이벤트 수집** | IoT Rule/Kinesis 등으로 디바이스 이벤트 수집 → Lambda 파싱·저장 → 대시보드·실시간 로그 연동 |

---

## 6. AWS 구성 요소

| 구성 요소 | 용도 |
|-----------|------|
| AWS IoT MQTT | 디바이스 ↔ 플랫폼 실시간 명령·상태 메시지 |
| AWS IoT Shadow | 디바이스 desired/reported 상태 및 업데이트 제어 |
| S3 | 펌웨어 바이너리 저장, 멀티파트 업로드 |
| CloudFront | 펌웨어 배포·다운로드 가속 및 캐싱 (TTL 설정) |
| Aurora (MySQL) | 제품·버전·고객·지사·업데이트 이력 등 관계형 데이터 |
| DocumentDB | 디바이스 메타, Shadow 스냅샷 등 문서형 데이터 |

| 구분 | 내용 |
|------|------|

| 공통 개발·환경 구성 | 1M (1명×1개월, 분석·설계와 병행) |
| 개발·테스트 | 4M (2명×2개월) |
| 합계 | 6M (최대 2명 투입 기준) |
| 비고 | 분석·설계 기간 동안 공통 개발 작업 1M이 병행 투입됨 |

## 8. 아키텍처 요약

```
[사용자/업체] → Cognito SSO → [FOTA 콘솔]
                                    ↓
[API Gateway] → [Lambda] ←→ Aurora + DocumentDB
     ↓
IoT MQTT / Shadow ←→ [디바이스]
     ↓
S3(펌웨어) ← CloudFront(배포, TTL)
```
