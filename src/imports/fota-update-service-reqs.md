# FOTA 업데이트 서비스 — 요구사항 및 Figma 요청용 정리

> 별도 페이지로 구성 예정. 기존 시스템 SSO(Cognito) 인증 연동.  
> Figma 디자인/와이어프레임 요청 시 참고용 문서.

---

## 1. 개요

- **서비스명**: FOTA(Firmware Over-The-Air) 업데이트 서비스
- **구성**: 기존 시스템과 분리된 **별도 페이지**로 제공
- **인증**: 기존 시스템 로그인 활용 — **AWS Cognito 기반 SSO** 처리

---

## 2. 인증·접근

| 항목 | 내용 |
|------|------|
| **인증 방식** | AWS Cognito SSO |
| **연동 범위** | 기존 시스템 로그인 처리 재사용 |
| **멀티 플랫폼** | 업체의 **다른 플랫폼**에서도 동일 Cognito로 접근 가능 (SSO) |
| **사용자 정보** | SSO 로그인 사용자 정보 조회·표시. 권한은 동일하게 간단하게(역할 구분 없음) |
| **디바이스 프로비저닝** | 별도 필요 — 디바이스 등록·인증 포함 |

---

## 3. AWS 구성 요소

| 구성 요소 | 용도 |
|-----------|------|
| **AWS IoT MQTT** | 디바이스 ↔ 플랫폼 실시간 명령·상태 메시지 |
| **AWS IoT Shadow** | 디바이스 desired/reported 상태 및 업데이트 제어 |
| **S3** | 펌웨어 바이너리 저장, **멀티파트 업로드**로 대용량 관리 |
| **CloudFront (CF)** | 펌웨어 배포·다운로드 가속 및 캐싱 (TTL 설정으로 유효기간 관리) |

### 3.1 펌웨어 다운로드 플로우

| 단계 | 처리 방식 |
|------|-----------|
| **1. 기본 다운로드** | CloudFront **TTL 설정**으로 캐시된 URL/리소스를 통해 디바이스가 펌웨어 다운로드 |
| **2. URL 만료 시** | 만료된 경우 디바이스가 **presigned URL**을 서버에 요청 |
| **3. 재발급** | 서버가 새 presigned URL 발급 → 디바이스가 해당 URL로 다운로드 재시도·이어받기 관리 |

- CF TTL로 트래픽·캐시 효율 확보, 만료 후에는 presigned URL 요청으로 보안·유효기간을 제어하는 방식으로 운영.

---

## 4. 데이터베이스

| DB | 용도 |
|----|------|
| **Amazon Aurora (MySQL)** | 관계형 데이터: 제품·버전·그룹·업체·지사·업데이트 이력·취소 등 |
| **Amazon DocumentDB** | 문서형 데이터: 디바이스 메타, Shadow 스냅샷, 유연한 스키마 데이터 |

---

## 5. 기능 요구사항

### 5.1 제품·버전 관리

- **제품별 버전 관리**: 제품 단위로 펌웨어 버전 등록·수정
- **펌웨어 업로드**: 제품별 펌웨어 파일을 **S3 멀티파트 업로드**로 관리 (대용량 지원)

### 5.2 업데이트 유형

| 유형 | 설명 |
|------|------|
| **제품 그룹별 업데이트** | 제품 그룹 지정 + 대상 디바이스 지정 후 업데이트 |
| **업체별·지사별·제품별 업데이트** | 업체별·제품별 업데이트 목록 지정 후 일괄 업데이트 |

### 5.3 업데이트 현황·제어

- **업체별·지사별·제품별** 업데이트 현황 조회·관리
- 진행 중 업데이트 **취소** 가능
- 기본적인 컴파일/부팅 오류 등 디바이스 단에서의 실패 시, **이전 정상 버전으로 자동 롤백** (디바이스 자체 보호 로직)
- 각 업데이트 Job 단위로 **이전 버전 복구(수동 롤백)** 기능 제공

---

## 6. 디바이스 프로비저닝

- 디바이스 등록·인증 정보 발급
- Cognito(SSO) 및 IoT 정책과 연동해 **다른 플랫폼에서의 접근** 지원
- 프로비저닝 플로우(등록 → 인증서/키 → Shadow 연동) 정의 필요

## 7. Figma 요청 시 포함할 요소 (체크리스트)

- [ ] **FOTA 전용 페이지** — 기존 시스템과 구분되는 레이아웃/네비게이션
- [ ] **SSO(Cognito) 로그인** — “기존 계정으로 로그인” 등 진입점
- [ ] **제품·버전 관리 화면** — 제품 목록, 버전 등록/수정, S3 멀티파트 업로드 진행 UI
- [ ] **업데이트 정책 설정** — 제품 그룹 지정, 디바이스 지정, 업체/지사/제품별 목록 지정
- [ ] **업데이트 현황 대시보드** — 업체/지사/제품별 진행률, 상태, 취소 버튼
- [ ] **디바이스 프로비저닝** — 디바이스 등록·목록·상태 화면
- [ ] **아키텍처 개요** — MQTT, Shadow, S3, CF, Aurora, DocumentDB 블록 다이어그램(선택)

---

## 8. 아키텍처 요약 (다이어그램용 한 줄)

```
[사용자/업체] → Cognito SSO → [FOTA 페이지]
                                    ↓
[FOTA 서비스] ←→ Aurora(MySQL) + DocumentDB
     ↓
IoT MQTT / Shadow ←→ [디바이스]
     ↓
S3(펌웨어) ← CF(배포, TTL)  // 만료 시 디바이스 → presigned URL 요청 → 재다운로드
```
